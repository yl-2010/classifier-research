# NoteLMs UI scratch

**Atelier · Blueprint · Orchard**

User flow (UI mock, no backend):

1. Paste notes or upload image → **Send**
2. Immediate **Received** confirmation
3. Classification + formatting happen off-screen (mocked delay)
4. Note appears in **Library** when ready
5. On the note, user can **manually change subject** (updates Research final label)

Users never see the three models or a confirm step.
